根据SQL文件的分析，**t_record_product表**和**t_project_product表**的关系如下：

---

## 一、表结构对比

| 表名 | 表说明 | 关键字段 |
|-----|--------|---------|
| **t_record_product** | 备案产品表 | F_Id, F_RecordProductName, F_Manufactor, F_RecordNo, F_BeginTime, F_EndTime |
| **t_project_product** | 项目建材产品表 | F_Id, F_ProjectId, F_ProductName, F_RecordNo, F_ManufacturerId, F_Quantity, F_UnitPrice |

---

## 二、表关系说明

### 1. 通过备案编号 (F_RecordNo) 关联

```
t_record_product (备案产品表)
       │
       │ F_RecordNo (备案编号)
       │ 1:N
       ▼
t_project_product (项目建材产品表)
       │
       │ F_RecordNo (备案编号)
```

**关系说明：**
- `t_record_product.F_RecordNo` = 备案产品的唯一备案编号
- `t_project_product.F_RecordNo` = 项目中使用产品的备案编号
- **一对多关系**：一个备案产品可以被多个工程项目使用

### 2. 业务逻辑关系

| 维度 | t_record_product | t_project_product |
|-----|-----------------|------------------|
| **数据性质** | 基础备案数据 | 业务使用数据 |
| **数据范围** | 全区/全市备案产品库 | 具体项目使用的产品 |
| **生命周期** | 备案有效期 (BeginTime~EndTime) | 项目周期内有效 |
| **使用场景** | 产品准入备案 | 项目建材填报 |

---

## 三、关联查询示例

```sql
-- 查询某备案产品在哪些项目中被使用
SELECT 
    rp.F_RecordNo AS 备案编号,
    rp.F_RecordProductName AS 备案产品名称,
    rp.F_Manufactor AS 生产厂家,
    pp.F_ProjectId AS 项目ID,
    p.F_ProjectName AS 项目名称,
    pp.F_Quantity AS 使用数量,
    pp.F_UnitPrice AS 单价
FROM t_record_product rp
LEFT JOIN t_project_product pp ON rp.F_RecordNo = pp.F_RecordNo
LEFT JOIN t_project p ON pp.F_ProjectId = p.F_Id
WHERE rp.F_RecordNo = '备案编号';
```

## 五、总结

| 关系类型 | 说明 |
|---------|------|
| **关联字段** | `F_RecordNo` (备案编号) |
| **关系类型** | 一对多 (1:N) |
| **业务含义** | 备案产品 → 项目使用产品 |
| **数据流向** | 备案库 → 项目填报 → 质量追溯 |

**核心逻辑：** 建材产品必须先完成备案（录入`t_record_product`），才能在工程项目中使用（录入`t_project_product`），通过备案编号实现产品溯源管理。
