# base_userrelation 用户关系表详解

## 1. 表结构

| 字段名 | 类型 | 说明 |
|-------|------|------|
| F_Id | varchar(50) | 主键ID |
| F_UserId | varchar(50) | 用户ID → 关联 base_user.F_Id |
| F_ObjectType | varchar(50) | 对象类型 (Role/Position) |
| F_ObjectId | varchar(50) | 对象ID (角色ID或岗位ID) |
| F_SortCode | bigint | 排序码 |
| F_CreatorTime | timestamp | 创建时间 |

## 2. 数据统计

- 总记录数: 5,220 条
- Role（角色）: 5,147 条
- Position（岗位）: 73 条

## 4. 作用说明

base_userrelation 是用户与角色/岗位的关联表，解决一个用户可能拥有多个角色或多个岗位的问题：

```
┌─────────────────────────────────────────────────────────┐
│                    base_userrelation                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   用户A ──┬──> 角色1 (项目经理)                          │
│           ├──> 角色2 (审核员)                            │
│           └──> 岗位1 (施工部经理)                         │
│                                                         │
│   用户B ──┬──> 角色1 (项目经理)                          │
│           └──> 岗位2 (技术员)                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 5. 与 base_user 的关系

在 base_user 表中，有以下字段存储用户关联信息：

- **F_RoleId** - 角色ID（text类型，可存多个，逗号分隔）
- **F_PositionId** - 岗位ID（text类型，可存多个，逗号分隔）
- **F_OrganizeId** - 组织ID（text类型，可存多个，逗号分隔）

base_userrelation 是对这些字段的更细致的扩展记录，用于：

1. **多对多关系**：一个用户可以有多个角色/岗位，一个角色/岗位可以分配给多个用户
2. **权限追溯**：通过此表可以追溯哪些用户拥有哪些角色/岗位
3. **审计追踪**：记录创建时间，方便审计

## 6. 总结

| 字段 | 作用 |
|-----|------|
| F_UserId | 哪个用户 |
| F_ObjectType | 是角色(Role)还是岗位(Position) |
| F_ObjectId | 具体是哪个角色或岗位 |

**简单理解**：这张表记录了"谁"拥有"什么角色"或"什么岗位"。
