## 六、数据流程说明

### 6.1 建材产品填报流程

```
1. 从一体化平台获取工程信息
   └─ jck_t_gc_sgxkz (施工许可证) → t_project (工程项目)

2. 填报建材产品信息
   └─ t_project_product (关联项目、生产企业、供应商)

3. 备案产品管理
   └─ t_record_product (产品备案，可关联jck_qyk_qyjcxx)

4. 质量追溯
   └─ t_quality_trace (关联产品，记录检测结果)
```

### 6.2 权限控制流程

```
1. 组织架构
   └─ base_organize (公司/部门)

2. 用户与组织关联
   └─ base_user.F_OrganizeId ↔ base_organize.F_Id

3. 岗位与组织关联
   └─ base_position.F_OrganizeId ↔ base_organize.F_Id

4. 用户角色/岗位分配
   └─ base_userrelation (F_ObjectType=role/position)

5. 权限分配
   └─ base_authorize (授权角色/用户/岗位访问菜单/按钮/列/表单/数据)
```

## 七、注意事项

1. **外键关联说明**：本系统未强制使用数据库外键约束，关联关系通过业务字段实现
2. **数据来源**：jck_\* 表为外部一体化平台数据接口表，仅供读取
3. **逻辑删除**：大部分表使用F_DeleteMark字段实现逻辑删除
4. **多值字段**：用户与组织/岗位/角色关联使用text字段存储多个ID（逗号分隔）
5. **配置文件**：数据库连接信息在application.yml中配置，当前为test环境
