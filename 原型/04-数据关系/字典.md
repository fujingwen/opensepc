## 五、迁移方案建议

### 5.1 数据库改造

1. **扩展 sys_dict_type 表**，添加树形支持：

   ```sql
   ALTER TABLE sys_dict_type ADD COLUMN is_tree char(1) DEFAULT '0' COMMENT '是否树形（Y/N）';
   ALTER TABLE sys_dict_type ADD COLUMN parent_id bigint DEFAULT 0 COMMENT '父级ID';
   ```

2. **扩展 sys_dict_data 表**，添加父级ID：

   ```sql
   ALTER TABLE sys_dict_data ADD COLUMN parent_id bigint DEFAULT 0 COMMENT '父级ID';
   ```

### 5.2 后端改造

1. **修改实体类**：
   - `SysDictType` 添加 `isTree`, `parentId` 字段
   - `SysDictData` 添加 `parentId` 字段

2. **修改 Service 层**：
   - 实现树形查询逻辑
   - 支持树形新增/修改/删除

3. **修改 Controller 层**：
   - 添加树形相关接口

### 5.3 前端改造

1. **修改字典类型页面**：
   - 添加"是否树形"开关
   - 支持树形展示

2. **修改字典数据页面**：
   - 添加树形选择器
   - 支持层级选择

### 5.4 数据迁移映射

| Test 字段 | Master 字段 | 说明 |
|----------|------------|------|
| F_FullName (type) | dict_name | 字典名称 |

## 六、注意事项

1. **主键策略**：Test 模式使用字符串主键，Master 模式使用自增主键，迁移时需要重新生成 ID

2. **树形结构**：Master 模式原生不支持树形字典，需要扩展表结构和代码

3. **数据量**：Test 模式有 38,537 条字典数据，迁移时需批量处理

4. **租户隔离**：Master 模式支持多租户，迁移时需考虑 tenant_id 字段

5. **编码规范**：Test 模式使用 F_EnCode 存储编码，Master 模式使用 dict_type + dict_value 组合

---

## 七、总结

| 项目 | 现状 | 建议 |
|-----|------|------|
| 表结构 | Test 模式字段更丰富 | 扩展 Master 模式表结构 |
| 树形支持 | Test 支持，Master 不支持 | 需扩展前后端代码 |
| 业务字典 | 38,537 条数据待迁移 | 编写迁移脚本 |
| 现有系统字典 | Master 有基础字典 | 保留，与业务字典共存 |
