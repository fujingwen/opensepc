# 代码回调清单

## 1. 目的

本清单只关注迁移功能清单中已经涉及的模块，用于识别当前代码、SQL 和 OpenSpec 文档里仍然沿用错误替代表的部分，作为后续回调到旧系统正确业务表的依据。

回调原则：
- 如果 `master` 已有正确承载表，则按 `database-rules.md` 把 `test` 数据迁入该承载表
- 如果 `master` 没有正确承载表，则直接在 `master` 建旧业务表并迁移旧表数据
- 不再继续围绕错误替代表推进实现

本次重点关注三类错误方向：
- `mat_project` 应回调为 `t_project`
- `mat_record` 不再作为真实承载表，备案产品统一围绕 `t_record_product`
- `mat_product` 应回调为 `t_project_product`

## 2. 工程项目模块

### 2.1 结论

- `test.t_project -> master.t_project`

当前错误方向：
- `master.mat_project`

- `construction-material-backend/hny-modules/hny-materials/src/main/resources/mapper/MatProjectMapper.xml`
  - 当前查询直接 `from mat_project`
  - 应改为围绕 `t_project`

后端实体与对象：
- `construction-material-backend/hny-modules/hny-materials/src/main/java/com/hny/materials/domain/MatProject.java`
  - 当前 `@TableName("mat_project")`
  - 应调整为 `t_project` 对应模型
- `construction-material-backend/hny-modules/hny-materials/src/main/java/com/hny/materials/domain/bo/MatProjectBo.java`
- `construction-material-backend/hny-modules/hny-materials/src/main/java/com/hny/materials/domain/vo/MatProjectVo.java`
  - 这些对象的字段语义仍围绕 `mat_project`
  - 后续应按 `t_project` 字段重新校对

### 2.3 需要回调的 OpenSpec 文档

- `openspec/changes/materials-project/design.md`
- `openspec/changes/materials-project/proposal.md`
- `openspec/changes/materials-project/tasks.md`

## 5. 质量追溯相关回调点

### 5.1 结论

质量追溯主表应为：
- `test.t_quality_trace -> master.t_quality_trace`

辅助关系表为：
- `t_product_relation`

但当前真实数据只能确认：
- `t_product_relation.F_ProductId -> t_project_product.F_Id` 成立

当前不能确认：
- `t_product_relation.F_CheckProductId -> t_quality_trace.F_Id`
- `t_product_relation.F_CheckProductId -> t_quality_trace.F_OriginalId`

### 5.2 需要纠偏的原型文档

- `openspec/原型/质量追溯/质量追溯.md`
- `openspec/原型/数据库关联关系/0.数据表关系.md`

这两份文档中，凡是把 `t_product_relation -> t_quality_trace` 写成“已被真实数据验证成立”的描述，都应改为：
- 这是设计目标链路
- 但当前 `test` 数据尚未验证命中
- 迁移实施前需要重新确认生成规则或补数规则

## 6. 优先级建议

建议代码回调顺序如下：

1. 先停掉 `mat_project` 和 `mat_product` 相关新增设计与 SQL
2. 先改后端 Mapper 和实体映射，使工程项目、建材填报回到 `t_project`、`t_project_product`
3. 再补 `master.t_project`、`master.t_quality_trace` 的数据库迁移脚本
4. 最后回写 OpenSpec 文档，统一删除错误替代表叙述
