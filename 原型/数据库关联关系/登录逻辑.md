# 建材监管系统 - 登录逻辑与权限体系详解

## 一、概述

本文档详细说明建材监管系统的用户登录逻辑，包括用户来源、登录方式、权限体系、表关系等。

---

## 二、登录入口

系统有两个登录入口，来自**数字住建一体化平台**：

| 登录入口 | 说明 |
|---------|------|
| 企业端登录 | 施工单位、生产企业、代理商用户 |
| 管理端登录 | 主管部门（住建局等）用户 |

登录地址：https://scxjsw.qingdao.gov.cn/jcmhqy/frame/pages/qymemmain/application_service/index

### 6.2 权限获取流程

```
用户登录
   │
   ├─> 1. 获取用户信息
   │      base_user.F_Id, F_Account, F_RealName
   │
   ├─> 2. 获取用户关联的组织/部门
   │      base_user.F_OrganizeId
   │      └─> base_organize.F_Id
   │
   ├─> 3. 获取用户关联的岗位
   │      base_user.F_PositionId
   │      └─> base_position.F_Id
   │
   ├─> 4. 获取用户关联的角色
   │      SELECT * FROM base_userrelation
   │      WHERE F_UserId = 用户ID AND F_ObjectType = 'Role'
   │      └─> 角色ID列表
   │
   ├─> 5. 获取菜单权限
   │      SELECT * FROM base_authorize
   │      WHERE F_ObjectId IN (角色ID列表)
   │      AND F_ItemType = 'module'
   │      └─> 可访问的菜单ID列表
   │
   ├─> 6. 获取按钮权限
   │      SELECT * FROM base_authorize
   │      WHERE F_ObjectId IN (角色ID列表)
   │      AND F_ItemType = 'button'
   │      └─> 可访问的按钮ID列表
   │
   └─> 7. 返回完整权限
          - 菜单列表
          - 按钮列表
          - 数据权限
```

```
权限层级：
1. 菜单权限 (module)   - 控制能看到哪些菜单
```

## 九、登录验证逻辑

### 9.1 验证流程

1. **接收登录请求**
   - 账号：手机号（F_MobilePhone）或账号（F_Account）
   - 密码：F_Password

2. **查询用户**

   ```sql
   SELECT * FROM base_user
   WHERE (F_MobilePhone = ? OR F_Account = ?)
   AND F_Password = ?
   AND F_EnabledMark = 1
   AND F_DeleteMark = 0
   ```

3. **验证成功**
   - 记录登录信息（F_LastLogTime, F_LastLogIP）
   - 生成Token
   - 返回用户信息

4. **获取完整权限**
   - 查询 base_userrelation 获取角色/岗位
   - 查询 base_organize 获取组织信息
   - 查询 base_position 获取岗位信息
   - 查询 base_authorize 获取权限列表

## 十、注意事项

1. **双写账号**：base_user 中同时存储了 F_Account 和 F_MobilePhone，两者都可以作为登录账号

2. **密码存储**：密码以明文或简单加密形式存储在 F_Password 字段

3. **逻辑删除**：通过 F_DeleteMark 字段实现逻辑删除，登录时需要检查

4. **启用状态**：通过 F_EnabledMark 字段控制用户是否可用

5. **一体化平台**：企业信息来源于数字住建一体化平台，同步到 t_companyinfo 表

6. **多值字段**：用户与组织/岗位/角色关联使用 text 字段存储多个ID（逗号分隔）
