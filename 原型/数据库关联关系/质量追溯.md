# 质量追溯模块分析

## 一、模块概述

质量追溯模块位于系统菜单 **质量追溯 (zlzs)** 下，包含四个子页面：

| 序号 | 页面名称 | 编码 | 路径 |
|-----|---------|------|------|
| 1 | 抽测缺陷建材产品 | spotTestingProduct | core/spotTestingProduct |
| 2 | 检测缺陷建材产品 | bhgcp | core/unqualifiedProduct |
| 3 | 缺陷建材使用情况 | bhgcpsyqk | core/unqualifiedUsage |
| 4 | 缺陷建材厂家 | bhgcj | core/unqualifiedManufacturer |

---

### 3.2 检测缺陷建材产品 (bhgcp)

对应同一张表 t_quality_trace，通过 F_DataStatus 或 F_ConclusionMark 等字段筛选出不合理的记录。

### 3.3 缺陷建材使用情况 (bhgcpsyqk)

可能涉及 t_quality_trace 与 t_product_relation 的关联查询，展示缺陷建材在项目中的使用情况。

### 3.4 缺陷建材厂家 (bhgcj)

根据 t_quality_trace 中的 F_FactoryName 字段进行分组统计，展示不合格产品的生产厂家。

## 四、缺陷建材使用情况页面分析

### 4.1 页面逻辑

**你的分析是正确的**，该页面的数据流程如下：

```
1. 从 t_quality_trace 获取有缺陷/不合格的检测记录
       │
       ▼
2. 通过 t_product_relation 关联找到对应的产品 (t_project_product)
   - t_product_relation.F_CheckProductId → t_quality_trace.F_Id
   - t_product_relation.F_ProductId → t_project_product.F_Id
       │
       ▼
3. 通过 t_project_product.F_ProjectId 找到对应的工程项目 (t_project)
       │
       ▼
4. 返回工程项目列表（包含产品信息和检测信息）
```

### 4.4 是否闭环字段

"是否闭环"字段可能的数据库来源：

| 字段名 | 表 | 说明 |
|-------|-----|------|
| F_CheckStatus | t_project_product | 检验状态（0=待检测，1=合格，2=不合格） |
| F_SupplierCheckStatus | t_project_product | 供应商检验状态 |

**闭环逻辑判断：**

- 当 F_CheckStatus = 1（合格）或 F_SupplierCheckStatus = 1（合格）时，表示"已闭环"
- 当 F_CheckStatus = 2（不合格）时，表示"未闭环"

## 五、注意事项

1. **时间戳转换**：数据库存储的是毫秒级时间戳，前端需要转换为日期格式显示

2. **结论字段**：F_Conclusion 存储的是文字描述结论，与 F_ConclusionMark（结论标志）不同

3. **有无对比数据**：hasCheckData 是前端计算字段，根据 t_product_relation 表中是否有对应产品的检测数据来判断

4. **复检状态**：recheckStatus 和 recheckRemark 字段可能存储在 t_quality_trace 表中，或作为前端计算字段

5. **闭环状态**：通过 t_project_product 表的 F_CheckStatus 或 F_SupplierCheckStatus 字段判断
