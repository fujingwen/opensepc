# 消息中心和通知公告模块分析

## 一、功能说明

### 1.1 消息中心功能

| 功能 | 说明 |
|-----|------|
| 发送消息 | 新建一条消息，选择接收人（用户/企业）进行发送 |
| 消息查看 | 查看别人发送给自己的消息 |
| 预警消息 | 查看系统发布的预警信息 |

### 1.2 通知公告功能

| 功能 | 说明 |
|-----|------|
| 发布公告 | 新建通知公告，无需选择接收人，所有人可见 |
| 库存信息公告 | 生产企业角色可发布 |
| 采购信息公告 | 施工企业角色可发布 |

---

## 二、涉及的数据表

| 表名 | 说明 | 记录数 |
|-----|------|-------|
| base_message | 系统消息/公告表 | 2 |
| base_messagereceive | 消息接收表 | 177,111 |
| t_message | 消息发送表（企业端） | 17 |

## 三、表结构

### 3.1 base_message 系统消息/公告表

| 字段名 | 类型 | 说明 |
|-------|------|------|
| F_Id | varchar(50) | 主键ID |
| F_Type | integer | 消息类型：1=消息，2=公告，3=预警 |
| F_Title | varchar(200) | 标题 |
| F_BodyText | text | 正文内容 |
| F_PriorityLevel | integer | 优先级 |
| F_ToUserIds | text | 发送给的用户ID列表（逗号分隔） |
| F_IsRead | integer | 是否已读 |
| F_Description | text | 描述 |
| F_EnabledMark | integer | 启用标志 |

### 3.2 base_messagereceive 消息接收表

| 字段名 | 类型 | 说明 |
|-------|------|------|
| F_Id | varchar(50) | 主键ID |
| F_Title | varchar(200) | 标题 |
| F_Content | varchar(500) | 内容 |
| F_Type | integer | 类型 |
| F_SendUser | varchar(50) | 发送人 |
| F_MessageId | varchar(50) | 消息ID → 关联base_message.F_Id |
| F_UserId | varchar(50) | 接收用户ID → 关联base_user.F_Id |
| F_IsRead | integer | 是否已读 |
| F_ReadTime | timestamp | 阅读时间 |
| F_ReadCount | integer | 阅读次数 |

### 3.3 t_message 消息发送表（企业端）

| 字段名 | 类型 | 说明 |
|-------|------|------|
| F_Id | varchar(50) | 主键ID |
| F_Title | varchar(200) | 标题 |
| F_ReceiverId | varchar(50) | 接收者ID → 关联t_companyinfo.F_Id（企业ID） |
| F_Content | text | 内容 |
| F_Appendix | text | 附件 |
| F_Status | integer | 状态 |
| F_EnabledMark | integer | 启用标志 |
| F_CreatorUserId | varchar(50) | 创建人ID |

## 七、业务流程

### 7.1 消息发送流程（消息中心）

```
用户发送消息
    ↓
选择接收人（企业）
    ↓
填写标题、内容、附件
    ↓
保存到 t_message 表
    ↓
F_ReceiverId = t_companyinfo.F_Id
    ↓
接收用户在消息查看中看到消息
```

### 7.2 公告发布流程（通知公告）

```
管理员/企业发布公告
    ↓
选择公告类型（公告/库存信息/采购信息）
    ↓
填写标题、内容
    ↓
保存到 base_message 表（F_Type=2）
    ↓
F_ToUserIds = 空（全员可见）
    ↓
所有用户都能在公告列表中看到
```

## 八、注意事项

1. **消息类型区分**：通过 base_message.F_Type 字段区分
   - F_Type = 1：消息（点对点发送）
   - F_Type = 2：公告（全员可见）
   - F_Type = 3：预警（特殊消息）

2. **接收人区别**：
   - t_message：接收人是企业（t_companyinfo）
   - base_messagereceive：接收人是用户（base_user）

3. **公告无接收人**：通知公告不需要选择接收人，发布后所有人可见

4. **角色限制**：库存信息只能由生产企业发布，采购信息只能由施工企业发布
