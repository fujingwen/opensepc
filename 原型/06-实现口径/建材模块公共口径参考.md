## 公共口径

### 1. 产品类别、产品名称、产品规格

- 查询场景默认复用 `ProductCascadeSelect`。
- 取值口径与建材产品页面 `materials/product` 保持一致。
- 产品类别使用 `sys_product` 类别节点 ID。
- 产品名称使用 `sys_product` 产品节点 ID。
- 产品规格使用 `productStandardSelect` 里的规格节点 ID。
- 如果页面是查询场景，不要重新手写一套类别/名称/规格联动。

### 2. 产品规格与单位

- 规格节点名称和单位的拆分规则统一走 `parseProductSpecAndUnit`。
- 规格名称展示统一走 `mapProductCascadeNode` 或 `parseProductSpecAndUnit`。
- 单位优先从规格节点名称中解析，不要在页面里重复写字符串拆分逻辑。
- 统计汇总中的总数量，如能确定唯一单位，则按“数量 + 空格 + 单位”展示，例如 `179061.44 ㎡`。

### 3. 生产单位省市区

- 生产企业原始区域字段来源于 `t_companyinfo.area`。
- 展示口径不是直接显示原始编码。
- 后端查询与下拉选项，应按基础数据行政区划表把编码转换成“省/市/区”文本。
- 前端如果拿到的是企业详情里的 `region`，优先使用 `region`。
- 前端如果只拿到 `area` 原值，统一走 `formatAreaCodes` 做兜底展示。

### 4. 质量监督机构

- 原始值可能是字典编码。
- 页面查询条件和表格展示都应显示字典标签，不直接显示 code。
- 后端统计查询口径统一按 `zljdjg` 字典转换后返回。

### 5. 统计分析页查询规则

- 产品类别、产品名称为必选项。
- 首次进入页面，未选择产品类别和产品名称前，不应自动查询列表。
- 合计行展示的是全量汇总，不受当前分页条数影响。
- 总数量、总金额统一保留两位小数。

### 6. 导出按钮处理原则

- 如果需求未完成，不要先接半成品导出逻辑。
- 可以先在按钮文案或点击行为中明确标记“待实现”，避免误导用户。

## 推荐复用方式

### 前端新增查询页时

1. 优先复用 `ProductCascadeSelect` 处理产品类别/名称/规格。
2. 优先复用 `materials.js` 中的公共工具，不在页面里重复实现：
   - `formatAreaCodes`
   - `parseProductSpecAndUnit`
   - `mapProductCascadeNode`
   - `formatMaterialDecimal`
   - `formatQuantityWithUnit`
3. 如果出现新的建材口径，先补到公共工具和本文档，再接入页面。

### 后端新增统计接口时

1. 产品类别/名称/规格的查询条件，优先对齐建材产品表现有 ID 口径。
2. 质量监督机构优先返回字典标签。
3. 生产单位省市区优先返回转换后的“省/市/区”文本。
4. 若返回汇总数据，同时返回数量单位，避免前端自行推断。

## 变更原则

- 公共口径发生变化时，先改公共代码，再改业务页面。
- 改完后同步更新本文档。
- 如果只是个别页面特殊，不要直接污染公共逻辑；先确认是否真的是模块级规则。
