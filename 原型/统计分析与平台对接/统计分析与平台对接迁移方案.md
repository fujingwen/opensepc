# 统计分析与平台对接迁移方案

## 1. 模块范围

本文件覆盖迁移清单中尚未承接但需要纳入规划的模块：

- 统计分析-基本统计分析
- 统计分析-采购价格统计分析
- 数据大屏
- 复检闭环
- 单点登录
- 数据共享
- 工程造价信息查询

## 2. 当前判断

### 2.1 统计分析

当前没有发现正式的统计分析模块页面或接口。

现有可作为统计基础的数据表主要是：

- `master.t_project_product`
- `master.t_record_product`
- 后续应补的 `master.t_project`
- 后续应补的 `master.t_quality_trace`

结论：统计分析还没有进入正式开发阶段，当前只能说“已有数据基础，缺统计层实现”。

### 2.2 数据大屏

仓库中没有发现现网可用的大屏模块代码，也没有新的菜单承载。

结论：数据大屏未实现，后续应建立在统计分析结果之上，而不是先做孤立大屏。

### 2.3 复检闭环

当前没有发现：

- 闭环业务表
- 闭环附件上传与预览接口
- 闭环页面

虽然旧分析文档里提到“复检状态”“是否闭环”等字段设想，但现有代码和 `master` 数据库中都没有正式实现。

结论：复检闭环未实现，且依赖质量追溯先落地。

### 2.4 单点登录

这一项已经具备基础实现：

- 后端：`HudAuthController`
- 前端：
  - `views/sso/qd_hudip/ssomanagement.vue`
  - `views/sso/qd_hudip/ssoenterprise.vue`

结论：单点登录属于“已实现基础能力，待联调验证”，不应再归类为待建设模块。

### 2.5 数据共享

当前没有发现面向住建数字一体化平台的数据共享控制器或接口。

结论：未实现，需要在核心主表稳定后再设计共享接口。

### 2.6 工程造价信息查询

当前没有发现公开查询页、控制器或专用数据表。

结论：未实现，需要单独设计功能边界和数据来源。

## 3. 与当前主线的依赖关系

### 3.1 统计分析依赖

统计分析必须建立在以下前提上：

1. `t_project` 落入 `master`
2. `t_project_product` 代码回调完成
3. `t_record_product` 差异补齐
4. `t_quality_trace` 落入 `master`

### 3.2 数据大屏依赖

数据大屏依赖统计分析指标口径先统一，否则会出现页面有图、口径不一致的问题。

### 3.3 复检闭环依赖

复检闭环依赖：

1. 质量追溯模块先落地
2. 缺陷产品与项目产品的关系规则先确认
3. 附件上传与预览能力接入

### 3.4 数据共享依赖

数据共享依赖主表和字段标准先稳定，不适合在 `mat_project/mat_product` 这类临时模型上开发。

## 4. 推荐实施顺序

### 4.1 先做数据主线

1. `t_project`
2. `t_project_product`
3. `t_record_product`
4. `t_quality_trace`

### 4.2 再做业务主线

1. 建材审核
2. 消息提醒
3. 质量追溯
4. 复检闭环

### 4.3 最后做展示与外联

1. 基本统计分析
2. 采购价格统计分析
3. 数据大屏
4. 数据共享
5. 工程造价信息查询
