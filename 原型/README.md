# 原型目录导航

当前目录按“总览、来源资料、业务模块、数据关系、问题跟踪、实现口径”六类内容整理，方便从不同视角查阅原型与迁移分析文档。

## 目录结构

- `01-总览`：迁移范围、模块状态、整体判断
- `02-来源资料`：操作手册、提取文本、流程对照、差异分析
- `03-业务模块`：按业务主题归类的原型说明
- `04-数据关系`：表关系、字段关系、登录逻辑、代码回调分析
- `05-问题跟踪`：历史问题、解决记录、未解决问题汇总
- `06-实现口径`：已经统一下来的开发约定、公共规则和复用口径

## 整理说明

- `02-来源资料` 保留原始手册、提取文本与对照分析，不删除历史来源文件。
- `03-业务模块` 保留业务主题分组，索引页只负责导航，不改动原业务文档结构。
- `04-数据关系` 统一使用更直观的文件名，去掉阶段性编号和临时命名。
- `05-问题跟踪` 将“问题 / 解决 / 未解决问题”拆分为更可读的历史记录与汇总入口。
- `06-实现口径` 用于沉淀已经按最终代码和联调结果统一下来的公共规则，不与业务原型和问题清单混放。
