# 产品管理

## 功能描述

产品管理模块用于管理系统中的建筑材料产品数据，采用三级树状结构：

- **产品类别**：顶级节点，如建筑钢材、防水材料、水泥等
- **产品名称**：二级节点，如螺纹钢、卷材、PO42.5水泥等
- **产品规格**：三级节点及以下，如规格:Φ12、强度:32.5等

## 数据来源

### 现有数据（从老系统同步）

产品相关数据已从 test 模式同步到 master 模式的字典表中：

| 表名 | 说明 |
|------|------|
| master.base_dictionarytype | 字典类型表 |
| master.base_dictionarydata | 字典数据表 |

### 数据层级关系

```
产品类别 (base_dictionarytype, is_tree=1, parent_id='271853493726217477')
├── 建筑钢材 (base_dictionarytype, is_tree=1)
│   ├── HPB300 (base_dictionarydata)
│   ├── 直径 (base_dictionarydata)
│   │   └── Ф4 (base_dictionarydata)
│   └── 牌号 (base_dictionarydata)
├── 防水材料 (base_dictionarytype, is_tree=1)
│   ├── 防水卷材
│   └── 防水涂料
├── 水泥 (base_dictionarytype, is_tree=1)
├── 预拌砂浆 (base_dictionarytype, is_tree=1)
└── ... (其他类别)
```

### 关键数据查询

```sql
-- 产品类别（材料名称）
SELECT id, full_name, en_code
FROM master.base_dictionarytype
WHERE parent_id = '271853493726217477' AND is_tree = 1
ORDER BY sort_code;

-- 产品名称（根据类别ID查询）
SELECT id, full_name, en_code
FROM master.base_dictionarytype
WHERE parent_id = '产品类别ID' AND is_tree = 1
ORDER BY sort_code;

-- 产品规格（根据产品名称ID查询）
SELECT id, full_name, en_code
FROM master.base_dictionarydata
WHERE dictionary_type_id = '产品名称ID'
ORDER BY sort_code;
```

## 推荐方案：新建产品表（sys_product）

系统管理模块下的产品管理，应使用 sys_ 前缀的表名。

### 表结构设计

```sql
-- ----------------------------
-- Table structure for sys_product
-- ----------------------------
DROP TABLE IF EXISTS "master"."sys_product";
CREATE TABLE "master"."sys_product" (
  "id" bigint NOT NULL DEFAULT nextval('master.seq_sys_product'),

  -- 树形字段
  "parent_id" bigint DEFAULT 0,              -- 父级ID
  "tree_path" varchar(500) DEFAULT '0',      -- 树形路径

  -- 类型区分
  "node_type" varchar(20) NOT NULL,          -- 节点类型：category/name/spec

  -- 业务字段
  "category_id" bigint,                      -- 所属产品类别ID（冗余）
  "full_name" varchar(200) NOT NULL,         -- 完整名称
  "en_code" varchar(100),                    -- 编码
  "specification" varchar(200),             -- 规格（仅spec类型使用）
  "unit" varchar(20),                       -- 单位（仅spec类型使用）
  "sort_code" int DEFAULT 0,                 -- 排序码
  "enabled_mark" tinyint DEFAULT 1,          -- 状态（0-停用 1-正常）
  "remark" varchar(500),                     -- 备注

  -- 审计字段
  "tenant_id" varchar(20),
  "create_dept" bigint,
  "create_by" varchar(50),
  "create_time" timestamp NOT NULL DEFAULT now(),
  "update_by" varchar(50),
  "update_time" timestamp NOT NULL DEFAULT now(),
  "del_flag" char(1) DEFAULT '0',

  PRIMARY KEY ("id")
);

-- 索引
CREATE INDEX idx_sys_product_parent_id ON "master"."sys_product"("parent_id");
CREATE INDEX idx_sys_product_tree_path ON "master"."sys_product"("tree_path");
CREATE INDEX idx_sys_product_node_type ON "master"."sys_product"("node_type");
CREATE INDEX idx_sys_product_category_id ON "master"."sys_product"("category_id");

-- 注释
COMMENT ON TABLE "master"."sys_product" IS '产品表';
COMMENT ON COLUMN "master"."sys_product"."id" IS '主键ID';
COMMENT ON COLUMN "master"."sys_product"."parent_id" IS '父级ID';
COMMENT ON COLUMN "master"."sys_product"."tree_path" IS '树形路径';
COMMENT ON COLUMN "master"."sys_product"."node_type" IS '节点类型(category-产品类别,name-产品名称,spec-产品规格)';
COMMENT ON COLUMN "master"."sys_product"."category_id" IS '所属产品类别ID';
COMMENT ON COLUMN "master"."sys_product"."full_name" IS '产品名称/类别名称';
COMMENT ON COLUMN "master"."sys_product"."en_code" IS '产品编码/类别编码';
COMMENT ON COLUMN "master"."sys_product"."specification" IS '产品规格';
COMMENT ON COLUMN "master"."sys_product"."unit" IS '单位';
COMMENT ON COLUMN "master"."sys_product"."sort_code" IS '排序码';
COMMENT ON COLUMN "master"."sys_product"."enabled_mark" IS '状态(0-停用,1-正常)';
COMMENT ON COLUMN "master"."sys_product"."remark" IS '备注';
```

### 数据层级说明

| 层级 | node_type | parent_id | category_id | 示例 |
|------|-----------|-----------|-------------|------|
| 产品类别 | category | 0 或 -1 | 自身ID | 建筑钢材 |
| 产品名称 | name | category的ID | category的ID | 螺纹钢 |
| 产品规格 | spec | name的ID | category的ID | 规格:Φ12 |

### 模块归属

- **所属模块**：系统管理（system）
- **表名前缀**：sys_
- **菜单位置**：系统管理 > 产品管理

## 替代方案：直接使用字典表

如果不想新建表，也可以直接使用现有的字典表进行管理：

- **产品类别**：master.base_dictionarytype 中 parent_id = '271853493726217477' 且 is_tree = 1
- **产品名称**：master.base_dictionarytype 中 parent_id 指向产品类别的记录
- **产品规格**：master.base_dictionarydata 中 dictionary_type_id 指向产品名称的记录

优点：无需新建表，直接利用现有数据
缺点：扩展性受限，无法添加额外业务字段

## 迁移建议

如果选择新建 sys_product 表，建议从现有字典表中导入初始数据：

```sql
-- 导入产品类别
INSERT INTO master.sys_product (id, parent_id, node_type, full_name, en_code, sort_code, enabled_mark)
SELECT
  nextval('master.seq_sys_product'),
  0,
  'category',
  full_name,
  en_code,
  sort_code,
  COALESCE(enabled_mark, 1)
FROM master.base_dictionarytype
WHERE parent_id = '271853493726217477' AND is_tree = 1;

-- 导入产品名称
-- 需要先获取产品类别的ID映射关系，然后导入
```

## 总结

| 方案 | 优点 | 缺点 |
|------|------|------|
| 新建 sys_product 表 | 扩展性强，可添加自定义字段，符合系统管理模块规范 | 需要迁移数据 |
| 使用字典表 | 无需新建表，利用现有数据 | 扩展性受限 |
