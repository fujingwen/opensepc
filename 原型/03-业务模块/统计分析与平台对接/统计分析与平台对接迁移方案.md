# 统计分析与平台对接迁移方案

## 1. 模块范围

本文件覆盖迁移清单中尚未承接但需要纳入规划的模块：

- 统计分析-基本统计分析
- 统计分析-采购价格统计分析
- 数据大屏
- 复检闭环
- 单点登录
- 数据共享
- 工程造价信息查询

## 2. 当前判断

### 2.1 统计分析

当前已经存在正式的统计分析模块页面和接口：

现有可作为统计基础的数据表主要是：

- `master.t_project_product`
- `master.t_record_product`
- `master.t_project`
- `master.t_quality_trace`

结论：统计分析已经进入正式开发阶段，当前重点不再是“有没有页面/接口”，而是：

- 继续校验统计口径
- 将“采购价格分析”持续标记为手册外增量能力
- 回写验收结论

### 2.2 数据大屏

仓库中已经存在数据大屏相关 change、表结构与聚合查询实现。

当前真实实现链路已经比较清晰：

- 前端独立路由：`/screen/dashboard`
- 前端页面：`construction-material-web/src/views/dashboard/materials/index.vue`
- 后端控制器：`MatDashboardController`
- 权限：`materials:dashboard:view`
- 页面按模块并行请求 5 个接口：
  - `GET /materials/dashboard/summary`
  - `GET /materials/dashboard/rates`
  - `GET /materials/dashboard/regions`
  - `GET /materials/dashboard/types`
  - `GET /materials/dashboard/lists`

当前大屏不是旧版 `test.blade_visual_*` 的 DataV 配置式方案，而是：

- 前端固定布局
- 后端拆分聚合接口
- 页面支持模块级降级
- 菜单统一映射到独立大屏路由

当前主要统计来源包括：

- `master.t_project`
- `master.t_project_product`
- `master.t_record_product`
- `master.t_quality_trace`

结论：数据大屏已实现基础能力，但本轮按用户要求未运行前端；当前仍需继续以统计口径稳定为前提做收尾。

### 2.3 复检闭环

当前没有发现：

- 闭环业务表
- 闭环附件上传与预览接口
- 闭环页面

虽然质量追溯页面已经存在，但“复检闭环”仍未形成独立稳定业务闭环。

结论：复检闭环未实现，且依赖质量追溯先落地。

### 2.4 单点登录

这一项已经具备基础实现：

- 后端：`HudAuthController`
- 前端：
  - `views/sso/qd_hudip/ssomanagement.vue`
  - `views/sso/qd_hudip/ssoenterprise.vue`

结论：单点登录属于“已实现基础能力，待联调验证”，不应再归类为待建设模块。

### 2.5 数据共享

当前没有发现面向住建数字一体化平台的数据共享控制器或接口。

结论：未实现，需要在核心主表稳定后再设计共享接口。

### 2.6 工程造价信息查询

当前没有发现公开查询页、控制器或专用数据表。

结论：未实现，需要单独设计功能边界和数据来源。

## 3. 与当前主线的依赖关系

### 3.1 统计分析依赖

统计分析必须建立在以下前提上：

1. `t_project` 落入 `master`
2. `t_project_product` 代码回调完成
3. `t_record_product` 差异补齐
4. `t_quality_trace` 落入 `master`

### 3.2 数据大屏依赖

数据大屏依赖统计分析指标口径先统一，否则会出现页面有图、口径不一致的问题。

### 3.3 复检闭环依赖

复检闭环依赖：

1. 质量追溯模块先落地
2. 缺陷产品与项目产品的关系规则先确认
3. 附件上传与预览能力接入

### 3.4 数据共享依赖

数据共享依赖主表和字段标准先稳定，不适合继续围绕历史临时模型开发。

## 4. 推荐实施顺序

### 4.1 先做数据主线

1. `t_project`
2. `t_project_product`
3. `t_record_product`
4. `t_quality_trace`

### 4.2 再做业务主线

1. 建材审核
2. 消息提醒
3. 质量追溯
4. 复检闭环

### 4.3 最后做展示与外联

1. 数据共享
2. 工程造价信息查询
