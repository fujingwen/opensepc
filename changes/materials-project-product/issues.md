# 建材产品问题记录

## 已解决问题

1. 建材产品 `/materials/product/list` 接口字段错误已修复。
2. 列表、查询条件、联动选择、表单布局、图片上传、状态流转等此前批量整理的问题已基本落地。

## 待处理问题

1. 列表操作栏的“重置信息确认状态”按钮及对应后端重置逻辑尚未实现。
2. 规格与单位的数据模型仍需继续确认：旧字典模型中的规格与单位并非严格结构化关联，当前基于 `master.sys_product.unit` 自动带出单位只是阶段性方案。

## 对照操作手册发现

以下问题为对照《青岛市建设工程材料信息管理平台操作手册》后新增确认：

1. 操作手册要求未审核记录即可编辑、删除；当前实现仅允许首轮“不通过”后的记录编辑/删除，限制偏严。
2. 操作手册允许“无备案证号”场景继续录入，并改为按生产单位补录；当前后端仍将 `recordNo` 作为新增必填项。
3. 操作手册要求“生产批号 / 生产日期”至少填写一项；当前实现未形成明确校验。
4. 操作手册要求导出数量上限为 2000 条；当前提案和实现均未限制导出上限。
5. 操作手册对供应商名称存在特殊业务口径，部分场景应展示“无”；当前实现虽有显示处理，但仍需继续核对录入、保存、导出链路是否完全一致。

## 已核对的数据事实

1. `master.t_project_product.has_record_no` 实际值为 `0/1/null`。
2. `master.t_project_product.check_status` 实际值为 `0/1/2/3/4`。
3. `master.t_project_product.supplier_check_status` 已存在且被历史数据使用。
4. `master.sys_dict_data` 中 `info_confirm_unit_type` 已存在。
