# 通知公告问题记录

## 问题清单

1. 库存信息发布和采购信息发布不应要求手工指定企业，应基于当前登录角色直接发布。

## 对照操作手册发现

以下问题为对照《青岛市建设工程材料信息管理平台操作手册》后新增确认：

1. 企业侧“系统公告”在操作手册中是只读查看能力，不应开放新增、编辑、删除；当前前后端和提案仍按可维护页面设计。
2. 操作手册要求库存信息发布、采购信息发布先保存为“草稿”，再执行“发布”动作变更为“已发布”；当前实现新增/修改时直接写入 `publish_status=published`，缺少草稿到发布的流转。
3. 操作手册要求库存/采购发布基于当前登录角色直接发布，不需要手工选择所属企业；当前页面和服务校验仍要求显式选择 `companyId/companyName`。
4. 操作手册对角色边界有明确约束：
   - 生产单位：查看系统公告、发布库存信息、查看采购信息
   - 施工单位：查看系统公告、查看库存信息、发布采购信息
   - 代理商：仅查看系统公告
   当前提案与实现仍按统一三页结构描述，尚未把角色差异固化成需求约束。

## 备注

1. 本文件中带“对照操作手册发现”的条目，均为按操作手册逐项比对后新增的问题来源标记。
