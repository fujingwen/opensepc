# 企业信息问题记录

## 已解决问题

1. 生产企业页面曾存在分页、联系人、联系电话、地区中文名显示异常等问题，现已修复。
2. `master.t_companyinfo.create_by` / `update_by` 曾错误写入 `admin`，现已改为实际登录用户 ID。
3. 生产企业、施工企业、代理商页面权限标识已与数据库真实权限对齐。
4. 代理商新增页“所属生产企业”已由手工输入改为下拉选择，并复用生产企业列表。
5. `getProductions` 接口已收敛为仅返回 `id` 和 `enterpriseName`。
6. 新增企业自动创建用户时的密码已改为加密落库。
7. 前后端联系电话校验已统一支持手机号和座机号。
8. `del_flag is null` 的历史数据已清洗并兼容查询。

## 已确认但暂不处理

1. 地区解析能力对历史脏数据的兜底仍不足，部分 `area` 值无法稳定映射到 `base_province` 标准地区名称。

## 对照操作手册发现

以下问题为对照《青岛市建设工程材料信息管理平台操作手册》后新增确认：

1. 操作手册侧重“企业完善自身信息”以及“生产单位创建本单位代理商”的业务流程；当前实现提供了施工企业、生产企业、代理商三类企业的后台主数据维护能力，范围明显大于操作手册描述。
2. 操作手册中的代理商维护应绑定当前生产单位上下文；当前实现允许从全量生产企业中选择所属企业，更接近后台管理模式，与手册中的企业自维护流程不完全一致。
