# 数据大屏规格

## ADDED Requirements

### Requirement: 数据大屏总览展示

系统 SHALL 提供建材数据大屏页面，并展示项目、建材填报、备案、企业和审核指标总览。

#### Scenario: 加载数据大屏首页

- **Given** 运行库中存在项目、建材填报、备案产品和缺陷建材数据
- **When** 用户进入 `数据大屏 / 建材数据大屏`
- **Then** 系统返回并展示项目总数、已填报项目总数、已填报建材产品总数、备案产品总数、施工企业数、生产单位数
- **Then** 系统展示信息确认率和审核通过率环形指标

### Requirement: 数据大屏区市地图

系统 SHALL 使用青岛区市边界地图展示各区市项目总数与参与项目数。

#### Scenario: 展示区市项目分布

- **Given** 运行库中存在项目地址与区市基础数据
- **When** 用户打开数据大屏地图区域
- **Then** 系统以 L7 地图渲染青岛区市边界
- **Then** 每个区市至少展示区市名称、项目总数和参与项目总数

### Requirement: 数据大屏统计图表

系统 SHALL 展示区市填报数量、产品类别数量和近 12 个月趋势。

#### Scenario: 展示统计图表

- **Given** 系统存在有效建材填报数据
- **When** 页面完成初始化
- **Then** 系统展示各区市建材产品填报数量图
- **Then** 系统展示各产品类别填报数量图
- **Then** 系统展示各产品类别近 12 个月填报趋势图

### Requirement: 数据大屏滚动榜单

系统 SHALL 展示最近项目填报审核列表与最近缺陷建材列表。

#### Scenario: 展示审核和缺陷榜单

- **Given** 系统存在最近填报记录和缺陷建材记录
- **When** 页面完成初始化
- **Then** 系统展示最近项目填报审核列表，至少包含项目名称、填报材料、备案证号、审核状态
- **Then** 系统展示最近缺陷建材列表，至少包含缺陷建材产品、批号、生产厂家、使用项目

### Requirement: 数据大屏运行库迁移

系统 SHALL 将数据大屏运行依赖迁回 `master` 模式，而不是直接依赖 `test` 模式。

#### Scenario: 迁移项目与缺陷建材运行表

- **Given** `test.t_project` 和 `test.t_quality_trace` 中已有可用数据
- **When** 运维执行本变更提供的迁移 SQL
- **Then** 系统创建 `master.t_project`
- **Then** 系统创建 `master.t_quality_trace`
- **Then** 系统将 `test.t_project` 和 `test.t_quality_trace` 的有效数据迁移至对应运行表
